# 演示登录指南

## 功能说明

由于后端 API 尚未实现，我们添加了一个**演示登录**功能，让您可以测试应用的所有功能。

## 如何使用

### 方式 1：演示账号登录（推荐）

1. 点击个人中心页面的"立即登录"按钮
2. 在登录模态框中，点击"使用演示账号登录"按钮
3. 页面会自动刷新，您将以演示账号登录

### 方式 2：手动输入（需要后端 API）

如果您已经配置了后端 API：

1. 输入邮箱和密码
2. 点击"登录"按钮
3. 系统会调用真实的 API 进行认证

## 退出登录

点击个人中心底部的"退出登录"按钮即可退出。

## 数据持久化

演示登录的数据存储在浏览器的 localStorage 中：
- `tarot_current_user`: 用户信息
- `tarot_auth_token`: 认证令牌

清除浏览器缓存会删除这些数据。

## 后续开发

当后端 API 准备就绪后：

1. 配置 `.env` 文件中的 `VITE_API_BASE_URL`
2. 后端应实现以下端点：
   - `POST /api/auth/login` - 用户登录
   - `POST /api/auth/register` - 用户注册
   - `POST /api/auth/logout` - 用户登出
   - `GET /api/user/profile` - 获取用户资料
   - 等等...

3. 移除演示登录功能，使用真实的 API 调用

## 技术实现

演示登录通过以下方式实现：

```typescript
// 创建演示用户对象
const demoUser = {
  id: 'demo-user-1',
  username: '神秘行者',
  // ... 其他属性
};

// 存储到 localStorage
localStorage.setItem('tarot_current_user', JSON.stringify(demoUser));
localStorage.setItem('tarot_auth_token', 'demo-token-' + Date.now());

// 刷新页面以触发 AuthContext 更新
window.location.reload();
```

## 注意事项

⚠️ **演示登录仅用于开发和测试**
- 不要在生产环境中使用
- 数据仅存储在本地，不会同步到服务器
- 刷新页面后数据仍然存在，直到手动退出登录或清除缓存
